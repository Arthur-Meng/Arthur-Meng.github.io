什么是ThreadLocal

  ThreadLoal变量是线程局部变量，同一个ThreadLocal所包含的对象，在不同的Thread中有不同的副本。这里有几点需要注意：

  • ThreadLocal提供了线程本地的实例。它与普通变量的区别在于，每个使用该变量的线程都会初始化一个完全独立的实例副本。
  • 因为每个Thread内有自己的实例副本，且该副本只能由当前Thread使用。这是也是ThreadLocal命名的由来。
  • 既然每个Thread有自己的实例副本，且其它Thread不可访问，那就不存在多线程间共享的问题。
  • ThreadLocal变量通常被private static修饰。当一个线程结束时，它所使用的所有ThreadLocal相对的实例副本都可被回收。
  • ThreadLocal是一个泛型类，保证可以接受任何类型的对象。

原理

  一个线程内可以存在多个ThreadLocal对象，所以其实是ThreadLocal内部维护了一个 Map ，这个Map不是直接使用的HashMap ，而是ThreadLoca 实现的一个叫做ThreadLocalMap的静态内部类。而我们使用的 get()、set() 方法其实都是调用了这个ThreadLocalMap类对应的 get()、set() 方法。

应用

  ThreadLoal提供了线程级别的信息存储，可以用于存储当前线程的重要标准和信息，并且可以全局共享，使得代码耦合度更低，且实现更优雅（尤其是和注解联合起来使用），例如：

  • 在接入层或者拦截器中将当前用户编号存入ThreadLoal，在之后的处理过程中都可以拿到当前用户编号
  • 通过注解将需要权限控制的标志存入ThreadLoal，在之后Mybatis的拦截器中根据标志判断是否需要进行权限过滤
  • 通过注解数据源信息存入ThreadLoal，在获取数据库连接时根据ThreadLoal中的数据源信息获取对应的连接
