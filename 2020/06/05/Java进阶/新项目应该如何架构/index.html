  最近由于工作的变动，会涉及到从零开发一个新项目，所以想简单分析下自己在构建新项目中的一些想法。

关于框架

  没什么好说的，如果没什么特殊情况的话，基本上是基于SpringBoot构建。推荐使用SpringBoot初始化构建（如官网构建或者IDEA构建），直接选择需要集成到项目里面的框架和依赖。

关于分层

接入层（Controller、Action或者微服务里面的接口层），只负责转发和记录请求

服务层（Service），负责具体的逻辑实现

持久化层（Dao，Mapper等），负责数据的持久化

  除了最上面的三层之后，还可能会抽出以下几个类和几层

  • 组件层（提供通用的组件服务，用于服务层调用）
  • 缓存层（提供常用数据的缓存，用于服务层调用）
  • 消息中间件层（提供消息的处理和推送，处理时调用服务层，被服务层调用推送消息）

关于日志

  • 使用Java日志门面（如Slf4j），方便后期切换日志框架（如log4j2和logback）
  • 合理规划级别，除了正常的info和error级别外，可以适量下debug级别的日志，方便后期遇到问题排查
  • 根据功能和类型的不同，可以输出到不同的日志文件
  • 如果对实时性没有要求，尽量使用异步日志模式而不是同步日志模式
  • 历史的日志文件建议根据年月日压缩归档

关于接口文档

  • 如果的mvc的项目，强烈建议使用Swagger2，不仅方便形成接口文档，还可以直接调用，利于测试和开发
  • 使用apiDoc生成形成接口文档，样式简洁，美观大方（至少比javadoc好看很多）
